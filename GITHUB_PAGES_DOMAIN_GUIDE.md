# Как разместить сайт на GitHub Pages и подключить домен

Эта инструкция для варианта без админки. Сайт будет статическим: он открывается по домену, но контент меняется только через правки в проекте и новый push.

## 1. Создать репозиторий у заказчика

В аккаунте заказчика на GitHub:

1. Нажать `New repository`.
2. Название можно сделать, например: `forum360-site`.
3. Выбрать `Public`.
4. Создать репозиторий.

Важно: для бесплатного GitHub Pages проще использовать публичный репозиторий.

## 2. Загрузить код сайта в репозиторий заказчика

Самый спокойный вариант:

1. Сначала мы пушим финальный код в репозиторий заказчика.
2. GitHub Actions сам соберёт сайт.
3. Ветка `gh-pages` появится автоматически после успешного workflow.

Если код переносится вручную через архив:

1. Скачать проект без `node_modules`, `dist`, `storage`, `.git`.
2. Загрузить файлы в новый репозиторий.
3. Убедиться, что в репозитории есть `.github/workflows/deploy.yml`.

## 3. Включить GitHub Pages

В репозитории:

1. Открыть `Settings`.
2. Слева выбрать `Pages`.
3. В блоке `Build and deployment` выбрать:
   - `Source`: `Deploy from a branch`;
   - `Branch`: `gh-pages`;
   - папка: `/ (root)`.
4. Нажать `Save`.

Если ветки `gh-pages` ещё нет, нужно дождаться завершения GitHub Actions после push в `main`.

## 4. Указать домен в GitHub

В `Settings → Pages`:

1. Найти поле `Custom domain`.
2. Ввести домен.

Так как домен кириллический, в GitHub лучше вводить техническую punycode-версию:

```text
xn--360-jedjmvf.xn--p1ai
```

Человеческий вид этого домена:

```text
форум360.рф
```

3. Нажать `Save`.

GitHub может проверять домен не сразу. Это нормально.

## 5. Настроить DNS у регистратора домена

Если домен должен открываться без `www`:

```text
форум360.рф
```

нужно добавить четыре A-записи:

```text
Тип: A
Имя: @
Значение: 185.199.108.153
```

```text
Тип: A
Имя: @
Значение: 185.199.109.153
```

```text
Тип: A
Имя: @
Значение: 185.199.110.153
```

```text
Тип: A
Имя: @
Значение: 185.199.111.153
```

Если нужен адрес с `www`:

```text
www.форум360.рф
```

добавить CNAME:

```text
Тип: CNAME
Имя: www
Значение: EkaterinaAlmaz.github.io
```

Итоговая запись:

```text
www  CNAME  EkaterinaAlmaz.github.io
```

Важно: в CNAME не нужно писать название репозитория. Только `username.github.io`.

## 6. Подождать обновления DNS

DNS может обновляться:

- иногда за 10-30 минут;
- иногда за несколько часов;
- редко до 24 часов.

Пока DNS обновляется, GitHub может показывать предупреждения в `Settings → Pages`. Это нормально.

## 7. Включить HTTPS

Когда GitHub увидит правильный DNS:

1. Открыть `Settings → Pages`.
2. Включить `Enforce HTTPS`.

Если галочка неактивна, подождать и обновить страницу позже.

## 8. Проверить сайт

Открыть:

```text
https://форум360.рф
```

Проверить:

- главную страницу;
- меню;
- кейсы;
- фотографии;
- модальные окна;
- мобильную версию.

## Важно про админку

Админки в этом варианте нет.

Почему:

- GitHub Pages хранит только статические файлы;
- он не запускает сервер;
- он не хранит базу данных;
- он не принимает загрузку фотографий.

Если заказчик потом захочет менять контент сам через админку, нужно будет возвращаться к варианту с VPS или отдельным backend-хостингом.

## Важно про пароль

Если пароль от GitHub передавался кому-то в переписке, лучше после настройки сайта сменить пароль и включить двухфакторную защиту.
